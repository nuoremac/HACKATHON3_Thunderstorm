# TWIST 02 : Gestion des Profils Incomplets

Ce document explique comment notre architecture gère les profils volontairement incomplets.

## L'Assumption Engine
Nous avons implémenté un système qui :
1. **Infère les intérêts** via le département académique.
2. **Estime les disponibilités** via l'emploi du temps théorique.
3. **Affiche des Cartes d'Assomptions** pour permettre une validation rapide par l'utilisateur.
